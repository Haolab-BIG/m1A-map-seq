# m1A-map-seq Pipeline

The m1A-MAP pipeline performs read preprocessing, 5вЂІ barcode removal, read mapping, PCR duplicate collapsing, mismatch counting, merging of untreated and demethylase-treated samples, candidate m1A site calling, filtering, reverse calling, and extraction of neighboring sequences for downstream analysis

Experimental Principleпјљ
m1A-MAP is a specialized short-read sequencing technology for base-resolution transcriptome-wide profiling of N1-methyladenosine (m1A), a key epitranscriptomic modification. It exploits m1A-induced base misincorporation during reverse transcription with TGIRT enzyme, combined with anti-m1A antibody enrichment of modified RNA fragments and AlkB demethylase treatment as a control. Via Illumina short-read sequencing, it identifies authentic m1A sites by quantifying mismatch rate differences between demethylated and untreated samples, enabling accurate detection of distinct m1A methylomes in nuclear- and mitochondrial-encoded transcripts.

```bash
# Reference Preparation

The original transcript FASTA contains duplicated sequence headers. If these duplicated headers are kept, BWA indexing and downstream SAM/BAM processing may fail because reference sequence names are expected to be unique.

To avoid this problem, de-duplicate the FASTA headers before building the index. The following script removes duplicated headers and keeps the first occurrence of each sequence ID, generating a cleaned reference file named `hg19_ncbiRefSeq_transcripts.fa`.

python3 - <<'PY'
seen = set()
infile = "/mnt/liuql/2_m1A-map/Reference/hg19_ucsc.fa"
outfile = "/mnt/liuql/2_m1A-map/Reference/hg19_ncbiRefSeq_transcripts.fa"

write_flag = False

with open(infile) as fin, open(outfile, "w") as fout:
    for line in fin:
        if line.startswith(">"):
            header = line[1:].rstrip("\n")
            name = header.split()[0]
            if name not in seen:
                seen.add(name)
                write_flag = True
                fout.write(line)
            else:
                write_flag = False
        else:
            if write_flag:
                fout.write(line)
PY
```

**Command Parameters**

**edit `config.yaml`**


- `samples`: Records the group assignment and file path for each sample. For single-end sequencing, only `R2` needs to be provided. You can add or remove sample entries as needed depending on the number of samples. Please list all control samples first, followed by all treatment/experimental samples.


- `threads`: Specifies the number of CPU threads used for multithreaded steps such as HISAT2 alignment.

- `fastqc_threads`: Specifies the number of CPU threads used for FastQC quality control.

- `preprocessing.sh`: Specifies the shell script used for raw read preprocessing, including adapter and quality trimming as well as 5вЂІ barcode removal.

- `collapse.sh`: Specifies the shell script used to collapse PCR duplicates based on mapping position, strand, and barcode information.

- `bam_to_mismatch_table.py`: Specifies the Python script used to parse deduplicated alignment files and generate per-position mismatch tables with base counts and mismatch rates.

- `merge_mismatch_tables.py`: Specifies the Python script used to merge mismatch tables from untreated and treated samples into pair-level comparison tables.

- `call_m1A_sites.py`: Specifies the Python script used to call candidate m1A sites from untreated-versus-treated mismatch comparison tables.

- `reverse_call_m1A_sites.py`: Specifies the Python script used to perform reverse calling for background or false-positive estimation.

- `combine_pair_calls.py`: Specifies the Python script used to combine candidate site calls from multiple sample pairs and retain sites supported by the required number of replicate pairs.

- `extract_neighbor_sequences.py`: Specifies the Python script used to extract neighboring reference sequences around called m1A sites for downstream sequence-context analysis.

- `min_base_quality`: Specifies the minimum base quality required for a nucleotide to be included in mismatch counting.

- `min_mapping_quality`: Specifies the minimum alignment quality required for a read to be included in mismatch counting.

- `fc_cutoff`: Specifies the minimum fold-change threshold in mismatch rate used for candidate m1A site calling.

- `diff_cutoff`: Specifies the minimum mismatch-rate difference threshold used for candidate m1A site calling.

- `min_mismatch_reads`: Specifies the minimum number of mismatch-supporting reads required for a site to be retained.

- `min_rep_support`: Specifies the minimum number of sample pairs supporting a site for it to be retained in the combined results.

- `flank_length`: Specifies the number of flanking nucleotides extracted on each side of a called site for neighboring-sequence analysis.
**run snakemake**

- `--use-singularity`: Enables execution of rules within a Singularity container to ensure a fully reproducible environment.
- `--singularity-args`: Allows passing additional arguments to the Singularity runtime (e.g., `--bind`, `--nv`, or custom options).
- `--cores`: Specifies the maximum number of CPU cores (threads) that Snakemake can use in parallel when executing workflow rules.
- `--bind`: Specifies the directories to be mounted within the Singularity container. Include all required paths such as raw data, scripts, container images, and references. The format is `/project_directory:/project_directory`. Multiple directories can be mounted by separating them with commas, for example: `/path1:/path1,/path2:/path2` (required)

## Output files

The pipeline writes all intermediate files and final results under `result/`, organized by analysis step as follows:

- `02_preprocessing/{sample}/barcodes.txt`: Contains the 5вЂІ barcode sequence extracted from each read.

- `03_mapping/{sample}/{sample}.sam.gz`: Contains the raw alignment results of processed reads against the reference.

- `04_dedup/{sample}/{sample}.collapsed.sam.gz`: Contains the deduplicated SAM output after collapsing reads with identical mapping position and barcode.

- `05_bam/{sample}/{sample}.collapsed.sorted.bam`: Contains the sorted deduplicated alignments for downstream mismatch analysis.

- `06_mismatch/{sample}/{sample}.mismatch.tsv`: Contains per-position base counts, mismatch counts, and mismatch rates for the sample.

- `07_merge_mismatch/pair_rep1.merged.tsv`: Contains the merged mismatch profile for one untreatedвЂ“treated sample pair.

- `08_calling/pair_rep1.m1A_sites.tsv`: Contains candidate m1A sites identified from one untreatedвЂ“treated sample pair.

- `08_calling/pair_rep1.reverse_sites.tsv`: Contains reverse-called sites from the same sample pair for background or false-positive estimation.

- `08_calling/m1A_sites.tsv`: Contains the combined final m1A site set retained after integrating pair-level calling results.

- `08_calling/reverse_called_sites.tsv`: Contains the combined reverse-called site set across all sample pairs.

- `09_motif/m1A_sites.fa`: Contains the flanking reference sequences extracted around the final called m1A sites.

- **`08_calling/m1A_sites.tsv`**
  - Comprehensive table of the final retained m1A candidate sites after combining results across sample pairs.
  - Key columns:
    - **`chrom`**: reference sequence name.
    - **`pos`**: 1-based position of the candidate site on the reference sequence.
    - **`ref_base`**: reference nucleotide at the candidate site.
    - **`support_pairs`**: number of sample pairs supporting this candidate site.
    - **`untreated_samples`**: untreated sample(s) supporting this site.
    - **`treated_samples`**: treated sample(s) supporting this site.
    - **`mean_depth_untreated`**: mean read depth in the untreated sample(s) across supporting pairs.
    - **`mean_mismatch_untreated`**: mean number of mismatch-supporting reads in the untreated sample(s) across supporting pairs.
    - **`mean_mismatch_rate_untreated`**: mean mismatch rate in the untreated sample(s) across supporting pairs.
    - **`mean_depth_treated`**: mean read depth in the treated sample(s) across supporting pairs.
    - **`mean_mismatch_treated`**: mean number of mismatch-supporting reads in the treated sample(s) across supporting pairs.
    - **`mean_mismatch_rate_treated`**: mean mismatch rate in the treated sample(s) across supporting pairs.
    - **`mean_diff`**: mean mismatch-rate difference between untreated and treated samples across supporting pairs.
    - **`mean_fc`**: mean mismatch-rate fold change between untreated and treated samples across supporting pairs.

- **`08_calling/pair_rep1.m1A_sites.tsv`**
  - Pair-level table of candidate m1A sites identified from one untreated-versus-treated sample pair.
  - Additional columns not described above:
    - **`depth_untreated`**: read depth at the site in the untreated sample.
    - **`mismatch_untreated`**: number of mismatch-supporting reads at the site in the untreated sample.
    - **`mismatch_rate_untreated`**: mismatch rate at the site in the untreated sample.
    - **`depth_treated`**: read depth at the site in the treated sample.
    - **`mismatch_treated`**: number of mismatch-supporting reads at the site in the treated sample.
    - **`mismatch_rate_treated`**: mismatch rate at the site in the treated sample.
    - **`diff`**: mismatch-rate difference between untreated and treated samples (`mismatch_rate_untreated - mismatch_rate_treated`).
    - **`fc`**: mismatch-rate fold change between untreated and treated samples (`mismatch_rate_untreated / mismatch_rate_treated`).
    - **`untreated_sample`**: untreated sample name.
    - **`treated_sample`**: treated sample name.
    - **`pair_support`**: support count contributed by this sample pair.

- **`08_calling/pair_rep1.reverse_sites.tsv`**
  - Pair-level table of reverse-called sites used for background or false-positive estimation from one untreated-versus-treated sample pair.
  - Additional columns not described above:
    - **`diff_reverse`**: reverse mismatch-rate difference (`mismatch_rate_treated - mismatch_rate_untreated`).
    - **`fc_reverse`**: reverse mismatch-rate fold change (`mismatch_rate_treated / mismatch_rate_untreated`).

- **`08_calling/reverse_called_sites.tsv`**
  - Comprehensive table of reverse-called sites retained after combining reverse-calling results across sample pairs.
